Produce community graph and node_id -> community mapping.

returns: UGraph reference community graph AND unordered_map reference node_id -> community mapping.

If this is the first time we have seen these nodes, add them to the nid_comm_map.

Approach: Since we ignore edges that connect 2 nodes that have a degree above the threshold, we know that these edges will define the connections between the communities that we are detecting. 2 possibilities: a) Only use edges that connect 2 nodes that have BOTH exceeded the threshold (gives us three classes of edge. transfer, null, and comm<-comm edges are static since both nodes are locked into a community) b) Use ALL edges that are ignored when making communities period (ALL null edges become comm edges) NOTE: this may lead to connecting communities that have dynamic definitions.

I will implement both a) and b) and see how it goes.
